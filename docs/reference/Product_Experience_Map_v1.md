#**Product UX Map v1**

**Дата:** 2026-03-09\
**Папка:**\
03 Product / UX Map

##**1. Purpose**

Product UX Map описывает:

- путь пользователя (user journey)

- ключевые сценарии использования

- экраны приложения

- взаимодействие с AI-гидом.

Документ используется:

- для разработки

- для дизайна

- для объяснения продукта инвесторам.

##**2. Core UX Principle**

Основной интерфейс Hey City — **голосовой диалог с AI-гидом**.

Пользователь **не ищет информацию вручную**.

Он разговаривает с гидом:

“Hey Sophia, what is this building?”

##**3. Primary User Journey**

###**Step 1**

**Install App**

Пользователь скачивает приложение.

Первый экран:

Welcome to **Hey City**

###**Step 2**

**Onboarding**

Приложение объясняет:

- как работает AI-гид

- как задавать вопросы

- что AI может рассказывать.

##**4. Guide Selection**

Пользователь выбирает гида.

Пример экрана:

Choose your guide.

Варианты:

- Sophia — History

- Marcus — Urban Stories

- Daniel — Architecture

- Elena — Art.

##**5. Voice Setup**

Пользователь разрешает:

- microphone

- location.

##**6. Home Screen**

Главный экран содержит:

- карту

- кнопку разговора

- текущего AI-гида.

**Interface Elements**

- Map

- Guide avatar

- Voice interaction button

- Mode selector.

##**7. Core Modes**

Пользователь может выбрать режим.

**Explore Mode**

AI рассказывает о местах вокруг.

Используется:

- при прогулке

- при поездке.

**AI Tour**

Пользователь говорит:

“Show me Art Deco buildings.”

AI создаёт маршрут.

**Guided Tours**

Библиотека готовых экскурсий.

**Creator Mode**

Создание собственного маршрута.

##**8. Proactive Interaction**

AI может сам начать разговор.

Например:

“You are approaching the Chrysler Building.”

Но если пользователь не реагирует —\
AI прекращает рассказ.

##**9. Conversation Mode**

Во время экскурсии пользователь может:

- задавать вопросы

- менять тему

- просить больше деталей.

Примеры:

“Tell me more.”

“Why was it built?”

“Who designed it?”

##**10. Adaptive Tour**

Экскурсия может изменяться.

Пользователь говорит:

“I only have 30 minutes left.”

AI меняет маршрут.

##**11. Look & Ask**

Пользователь может спросить:

“What am I looking at?”

AI определяет объект.

##**12. Personal City History**

Приложение сохраняет:

- места

- маршруты

- темы.

Экран:

My City History.

##**13. Saved Stories**

Пользователь может сохранить объект.

Кнопка:

Save for later.

##**14. Notifications**

AI может предложить:

“There is a famous gallery nearby.”

##**15. Settings**

Настройки включают:

- выбор гида

- голос

- язык

- интересы

- история.

##**16. Safety Mode (Vehicle)**

Во время движения:

- минимальный экран

- аудио-режим.

##**17. Creator Mode UX**

Создание маршрута:

Step 1\
Add stop

Step 2\
Select POI

Step 3\
Generate story (AI)

Step 4\
Edit

Step 5\
Publish.

##**18. Social Layer (future)**

Пользователь может:

- делиться маршрутами

- подписываться на гидов.

##**19. UX Philosophy**

Hey City стремится к:

- минимальному интерфейсу

- голосовому взаимодействию

- ненавязчивому AI.

##**20. Key UX Principle**

Пользователь не должен:

- читать длинные тексты

- нажимать много кнопок.

Основной опыт — **разговор с городом**.
